Rabin-Karp Algorithm Explanation

Simple and structured view of how hashing based string matching detects fraud patterns efficiently.

1. Problem Statement

The detector must search a long message for many suspicious words and phrases such as account warnings, payment requests, and verification prompts. Rechecking every character for every keyword can be expensive.

2. Why Rabin-Karp

Rabin-Karp converts text windows and pattern strings into numeric hash values. Instead of comparing complete strings at every position, it compares hashes first. Full character comparison is only done when hashes match. This reduces repeated direct comparisons in practical scanning workloads.

3. Step by Step Process

  1. Choose a base and prime modulus for hashing.
  2. Compute hash of the target pattern.
  3. Compute hash of the first window of text with same length as pattern.
  4. Compare window hash and pattern hash.
  5. If hashes match, verify characters one by one to avoid collision errors.
  6. Slide the window by one position and update hash using rolling hash formula.
  7. Repeat until the window reaches the end of text.

4. Time Complexity

Average case for one pattern is O(n + m), where n is text length and m is pattern length, because rolling hash update is constant time per shift. Worst case can reach O(nm) when many hash collisions occur, but this is uncommon with a good modulus and input distribution.
