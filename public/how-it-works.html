How the Detection Workflow Operates

From user input to risk scoring and highlighted output.

Step 1: Message Ingestion

A user submits or pastes a potentially suspicious UPI related message in the detector page. Real-time monitoring starts immediately as text changes.

Step 2: Pattern Scanning

The system scans the text against a broad list of known scam words and phrases. Matching is performed with Rabin-Karp rolling hash based search.

Step 3: Match Validation

Hash collisions are handled by character verification to ensure each detected phrase is a true match. This avoids false positives caused by equal hash values alone.

Step 4: Risk Output

The interface highlights suspicious regions in the original text, lists found phrases with frequencies, and calculates a fraud probability score to support decision making.

Real World Relevance

  • Supports rapid initial triage for support and fraud teams.
  • Improves user awareness when handling urgent payment messages.
  • Can be used as a pre-filter before escalation to advanced risk systems.
  • Provides transparent rule-driven outcomes suitable for demonstrations and audits.

What To Do If Risk Is High

  1. Do not click any link or approve collect requests mentioned in the message.
  2. Verify the sender using the official app, website, or customer support number.
  3. Never share OTP, PIN, password, CVV, or card details with anyone.
  4. Report suspicious messages to your bank and cybercrime portal immediately.

Important Limitation

This detector is keyword and pattern based. It helps with early screening, but should be combined with human verification and trusted channels before taking any payment decision.
